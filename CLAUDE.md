This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Wezar is a **static-site luxury jewelry e-commerce** storefront ‚Äî zero build tools, zero dependencies, pure HTML/CSS/JS. No npm, no bundler, no compilation step.

## Development

Open any `.html` file directly in a browser, or serve locally:

```bash
python3 -m http.server 8000
# or
npx serve .
```

No build, lint, or test commands exist in this project.

## Architecture

### Data Layer ‚Äî `assets/data.js`

The single source of truth for all runtime state. Import it on every page.

- **`PRODUCTS`** ‚Äî array of 16 product objects (`id`, `name`, `collection`, `category`, `price`, `tag`, `edition`)
- **`COLLECTIONS`** ‚Äî array of 4 collection objects with `tone` ('dark' | 'pearl' | 'blush' | 'deep')
- **`cart`** ‚Äî singleton persisted to `localStorage` key `wezar.cart.v1`; shape: `[{ id, qty }]`
  - API: `.read()`, `.write()`, `.add(id)`, `.update(id, qty)`, `.remove(id)`, `.count()`, `.subtotal()`
  - Reactive: `.add()` / `.update()` call `._notify()` which updates every `[data-bag-count]` element on the page

### Page Templates

`data.js` also exports `injectChrome(page)` which injects the shared nav + footer into every page. Call it with the current page name (e.g. `'shop'`) to apply the active nav state. Header is rendered by `renderHeader(page)`, footer by `renderFooter()`.

### Product Cards & Procedural Art

- `productCard(product)` ‚Äî renders a product card HTML string used in grids
- `productArt(product)` ‚Äî generates inline SVG line art based on `product.category` + a seed; stroke color varies by collection tone

### Routing / Filtering

All inter-page navigation uses query strings:
- `?collection=heirloom` ‚Äî filter by collection
- `?category=rings` ‚Äî filter by category
- `?id=p01` ‚Äî open a specific product detail page

Pages read `new URLSearchParams(location.search)` and filter `PRODUCTS` client-side.

### CSS Architecture

Two stylesheets, always loaded in order:

1. **`wezar.css`** ‚Äî design tokens only: CSS custom properties for color, typography, spacing, motion, and semantic utility classes (`.wz-display`, `.wz-caption`, `.wz-label`, `.bg-onyx`, etc.)
2. **`site.css`** ‚Äî component library built on top of those tokens: nav, product grid, cards, buttons, forms, footer

Key tokens:
- Brand colors: `--wz-rose` (#F0A4B1), `--wz-champagne` (#C9A875), `--wz-onyx` (#0A0A0A), `--wz-pearl` (#F4EFE8)
- Spacing scale: `--wz-space-1` (4px) ‚Üí `--wz-space-11` (192px)
- Signature easing: `cubic-bezier(0.22, 0.61, 0.36, 1)`

### `<image-slot>` Web Component ‚Äî `assets/image-slot.js`

A self-contained custom element for user-uploadable product images.
- Accepts PNG, JPEG, WebP, AVIF (no SVG/GIF)
- Downscales to WebP at q=0.85, max 1200px
- Supports pan/zoom/reframe in cover mode
- Persists image data-URLs + transform state to `.image-slots.state.json` sidecar (server-side write needed for persistence)

## Animation ‚Äî `assets/motion.js`

Framer-motion (DOM/vanilla JS subset) loaded via CDN. Add to any page that needs animation:

```html
<script type="module" src="assets/motion.js"></script>
```

Then use `window.Motion.*` in regular scripts:

```js
const { animate, scroll, inView, stagger } = window.Motion;

// Fade-in on page load
animate('.hero', { opacity: [0, 1], y: [20, 0] }, { duration: 0.6, easing: 'ease-out' });

// Scroll-driven parallax
scroll(animate('.hero-bg', { y: [0, 80] }));

// Trigger when element enters viewport
inView('.product-card', ({ target }) =>
  animate(target, { opacity: [0, 1], y: [16, 0] }, { duration: 0.4 })
);
```

Available exports: `animate`, `scroll`, `inView`, `timeline`, `stagger`, `spring`, `easeIn`, `easeOut`, `easeInOut`.

## Known Gaps

- **No payment processing** ‚Äî checkout is UI-only
- **No backend** ‚Äî products/orders are hardcoded in `data.js`
- **No real product images** ‚Äî placeholder SVGs generated by `productArt()`
- **No newsletter integration** ‚Äî form submits are no-ops
